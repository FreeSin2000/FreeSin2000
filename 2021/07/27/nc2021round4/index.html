B Sample Game

基本题意：

​ 游戏每轮随机取数，\([1-n]\)的每个数都有一定概率被取到。

​ 如果最近一轮取得的数不是当前已经取得所有数的最大值结束游戏，否则可以继续随机取数，求游戏的轮数平方的期望。

场上思路：

​ 我: 定义\(E_x(y)\)和\(E_{x^2}(y)\)表示如果可以选中\(y\)并且还可以继续游戏的轮数的期望以及轮数平方的期望，并且结合\(E((x+1) ^2) = E(x^2) +2E(x) + 1\)想出了一个假递推式: \[ E_{x}(y) = p(y)\sum\limits_{j = 1}^{y}E_x(y) +1 \\ E_{x^2}(y) = p(y)\sum\limits_{j = 1}^{y}(E_{x^2}(y)+2E_{x}(y)+ 1) \]

​ 真的很可笑，这显然是不对的（无论是表示还是递推式），说是推式子，其实只是凭印象乱搞，然而但是由于我概率论学的并不好并没有及时看出来。

​ 由于我并不了解随机过程并不能很好的表达，大概就是定义的状态对应的期望并不是一个最终状态，所以并不容易表示，对这样的问题用概率描述要比用期望容易一些。后面我在用级数推边界的时候发现和上述式子的封闭形式不同才意识到自己的错误。

​ 另外一点就是我能想到用级数去推一个特殊情况但是最终竟然没想到用生成函数，也是十分迷人。因为在比赛中我想过转化问题等价为寻找一个非递减的方案的概率并也想到了用生成函数来计算，但是场上我把自己假了，我竟然认为多项式乘起来会失序，很迷，很可惜题解就是这样做的。

正确思路：

​ 场上想的差不多，需要维护的是一个还能继续游戏的状态形式，定义如下 \[ P(len\ge i) \] ​ 表示游戏轮数大于\(i\)的概率，于是我们所要求的就可以表示为 \[ \sum\limits_{i = 0}^{\infty}(P(len\ge i) - P(len\ge i+1)) \times (i +1) ^2 \] ​ 这里有个常规操作，如果是这样的一类问题我们可以用未结束的概率的差来获得结束的概率，记得2021cccc天梯赛L3最后一题也是用的这样的套路。

​ 进一步化简 \[ =\sum\limits_{i = 0}^{\infty}P(len\ge i)\times((i +1)^2-i^2) \] ​ 对于无穷多项数列的常规操作，加减的项重新分组即可，展开 \[ =\sum\limits_{i = 0}^{\infty}(2iP(len\ge i) +P(len \ge i))\tag{1} \] ​ 然后就是第二点，场上想到了但是没有完全想到的利用处理无穷的方法，我们定义生成函数 \[ f(x) = \sum_{i = 0}^{\infty} P(len\ge i) \times x^i\tag{2} \] ​ 求\(P(len\ge i)\)​等价于游戏中取的一个长度为\(i\)的非递减序列的概率，大概就是一个形如 \[ \sum\limits_{\forall x,y\ x <y\ q_x \le q_y}\prod\limits_{i=1}^{i}p(q_i) \] ​ 对于\(j\)的概率\(p_j\)，我们构造如下的多项式 \[ \sum\limits_{j = 0}^{\infty} p_jl\times x^l\tag{3} \] ​ 显然如果我们把每个数的上述多项式\((3)\)​​​​​相乘，显然就能得到上述\(f(x)\)​​​即公式\((2)\)​​​​​​，多项式乘法得到的\(x^i\)​项的系数就是我们要求的选\(i\)个数非递减排列的概率之和，利用等比数列求和公式可以把公式\((3)\)​进一步简化，最终得到 \[ f(x) = \prod\limits_{i = 1}^{n}\frac{1}{1-p_ix}\tag{4} \] ​ 观察式\((1)\)发现等于 \[ 2f^{'}(1)+f(1) \] ​ 于是\(O(n)\)​求解即可。
